Power Query Data Cleaning Walkthrough

How to reliably parse space-delimited pasted text with multi-word fields into five clean columns — Customer, Product, Region, Amount, Date — using Power Query in Excel.

01 — Scenario

The Scenario & Sample Data

A colleague sends a plain-text report. You copy it and paste it into Excel. Each row represents a sale, but because Customer names and Product names can each contain a space (e.g. John Smith, Whey Powder), you cannot split on every space blindly. The five fields you need are:

Customer Product Region Amount Date

Raw pasted text (space-delimited, inconsistent spacing)

John Smith     Whey Powder    North   150.00   2024-01-15
Alice Brown    Protein Shake  East    89.99    2024-01-16
Bob Johnson    BCAAs          South   45.50    2024-01-17
Diana Prince   Whey Powder    West    150.00   2024-01-18
Alice Brown    BCAAs          North   45.50    2024-01-19
               <blank row>
(blank)        Protein Shake  East    89.99    2024-01-20
⚠ Key challenge
Both Customer (e.g. "John Smith") and Product (e.g. "Whey Powder") may be two words separated by a single space — so naively splitting on every space produces too many columns. The strategy is to anchor on the known fields: Region is always one of {North, South, East, West}, Amount is always numeric, and Date is always in a date format.

Target output (5 clean columns)

John SmithWhey PowderNorth150.002024-01-15
Alice BrownProtein ShakeEast89.992024-01-16
Bob JohnsonBCAAsSouth45.502024-01-17
Diana PrinceWhey PowderWest150.002024-01-18
Alice BrownBCAAsNorth45.502024-01-19
02 — Excel Setup

Step 1 — Paste & Load into Power Query

Do this regardless of which approach (A or B) you follow next. The goal is to get your pasted data into the Power Query Editor.

  1. Paste the text. Click an empty cell (e.g. A1), then press Ctrl+V. Excel may paste as a single column or as multiple columns — note which case you get. (See Approach A vs B below.)

  2. Select your data range. Click anywhere inside the pasted data. If the data is a single column, select that column from the first row to the last non-empty row.

  3. Convert to an Excel Table. Press Ctrl+T. In the "Create Table" dialog, confirm the range and tick "My table has headers" if the first row is a header — otherwise leave it unchecked. Click OK.

  4. Open Power Query. Go to Data tab → Get & Transform Data group → click From Table/Range. The Power Query Editor window opens with your data loaded.

ℹ Note — "From Table/Range" vs other connectors
"From Table/Range" reads the Excel table you just created directly in memory — no file path needed. All transformations are recorded as steps you can edit at any time in the Applied Steps pane on the right.

04 — Approach A

Step 2A — Paste Created Tab-Separated Columns

If Excel correctly created 5 (or more) columns when you pasted, the heavy lifting is already done. You just need to verify the delimiter, clean up header names, and set data types.

Verify delimiter & promote headers

  1. Check column count. In the Power Query Editor, confirm you see exactly 5 columns in the preview. If you see more than 5, extra splits happened — use Remove Columns to drop unwanted ones or merge over-split fields (see Approach B for merge technique).

  2. Promote first row to headers (if needed). If the column names are Column1, Column2… rather than meaningful names, go to Home tab → Use First Row as Headers.

  3. Rename columns manually if needed. Double-click each column header in the preview and rename to: Customer, Product, Region, Amount, Date.

Set column data types

  1. Customer & Product — Text. Click the type icon (top-left of the column header, showing ABC or a number icon) and choose Text.

  2. Region — Text. Same as above — set to Text. Optionally: Transform tab → Format → Capitalize Each Word to normalise casing.

  3. Amount — Decimal Number. Click the type icon → Decimal Number. If values contain currency symbols (£, $), first apply Transform → Replace Values to remove the symbol before changing the type.

  4. Date — Date. Click the type icon → Date. If Power Query shows a type-conversion error, use Transform → Data Type → Using Locale and specify the correct locale (e.g. English (United Kingdom) for DD/MM/YYYY).

✔ After Approach A
Your 5 columns should now have the correct names and types. Skip to Step 3 — Clean Up.
05 — Approach B

Step 2B — Paste Landed as a Single Column

This is the harder case. Every row is one long string like:
John Smith    Whey Powder   North  150.00  2024-01-15

The strategy: split aggressively on whitespace to get tokens, then use the known structure of the last three fields (Region = one of {North|South|East|West}, Amount = numeric, Date = date pattern) to anchor the split — leaving the remaining tokens to reconstruct Customer and Product.

Split column by whitespace delimiter

  1. Select the raw column. Click the column header to select it.

  2. Open Split Column dialog. Go to Home tab → Split Column → By Delimiter.

  3. Configure the split. In the dialog:

    • Delimiter: choose Space (or type a single space in the custom field)
    • Split at: Each occurrence of the delimiter
    • Expand Advanced options → tick "Split into Columns"
    • Click OK

  4. Handle extra blank columns from multiple spaces. Power Query creates many extra columns for each consecutive space. Select all the empty/null columns in the preview, right-click → Remove Columns. Alternatively, use Home → Remove Columns → Remove Other Columns after selecting only the non-empty token columns.

ℹ Tip — Split by multiple spaces at once (M formula)
To avoid dozens of blank columns from consecutive spaces, use a custom M step instead. Click the fx formula bar and replace the split step with:

= Table.SplitColumn(#"Previous Step", "RawData", Splitter.SplitTextByAnyDelimiter({" ", " "}, QuoteStyle.None))

Or, even cleaner — first Trim the column (Transform → Format → Trim) to collapse runs of spaces, then split on a single space.

Identify the anchor tokens (Region, Amount, Date)

After splitting you have columns like Token.1, Token.2 … Token.N. The last three tokens in each row are always Date, Amount, Region (reading right-to-left), regardless of how many words are in the Customer or Product names.

  1. Rename the last three token columns. Scroll to the rightmost token columns. Double-click and rename: last column → Date, second-to-last → Amount, third-to-last → Region.

  2. Verify Region values. Click the Region column drop-down filter and confirm values are only North, South, East, West (no stray words). If a row's Region column contains a non-region word, your token count is off for that row — see the troubleshooting note below.

  3. Verify Amount values. Set Amount column type to Decimal Number. Rows with type errors indicate the token split is misaligned for those rows.

⚠ Troubleshooting misaligned rows
If some rows have the wrong value in the Region column, it usually means one of the Customer or Product names has an unexpected number of words. Fix by adding a Conditional Column (Add Column → Conditional Column) that checks whether the candidate Region token is in {"North","South","East","West"}; if not, shift your token window by one.

Recombine remaining tokens into Customer and Product

The remaining columns to the left of Region are the Customer and Product tokens. With both names being exactly two words each, this is columns 1–2 (Customer) and columns 3–4 (Product). If field widths vary, use the rule-based approach below.

When Customer and Product are always exactly two words each

  1. Merge Customer tokens. Select Token.1 and Token.2 (hold Ctrl and click both). Go to Add Column tab → Merge Columns. Separator: Space. New column name: Customer. Click OK.

  2. Merge Product tokens. Select Token.3 and Token.4. Repeat Merge Columns with separator Space. Name: Product.

  3. Remove the original token columns. Select all remaining Token.N columns, right-click → Remove Columns.

Rule-based approach — when field widths are variable

If Customer might be one or three words and Product might be one or two words, use a custom M column to find the Product boundary using the known Region anchor: 

// Custom Column: extract everything left of Region as a combined string
// Add Column → Custom Column → paste this formula:

let
    rawTokens  = Text.Split(Text.Trim([RawData]), " "),
    // known anchors: last 3 tokens are Region, Amount, Date
    numTokens  = List.Count(rawTokens),
    leftTokens = List.FirstN(rawTokens, numTokens - 3),
    // find the Product boundary: scan left-to-right, Product ends just before Region
    // heuristic: Product = last two tokens before Region (adjust if needed)
    prodStart  = List.Count(leftTokens) - 2,
    custTokens = List.FirstN(leftTokens, prodStart),
    prodTokens = List.LastN(leftTokens, 2)
in
    [ Customer = Text.Combine(custTokens, " "),
      Product  = Text.Combine(prodTokens, " ") ]
ℹ Extracting the Record fields
The formula above returns a Record. After adding it as a custom column, click the expand icon (⇲) in the new column header and select Customer and Product to unpack them into two columns.

An alternative GUI-only approach using the Extract feature:

  1. Add a helper "LeftOfRegion" column. Add Column → Extract → Text Before Delimiter. Delimiter: the Region value (e.g. North with a leading space). Because the Region is different per row, use a Custom Column formula instead: Text.BeforeDelimiter([RawData], " " & [Region]). Name it CustomerProduct.

  2. Split CustomerProduct by the last space. Select CustomerProduct → Split Column → By Number of Characters — or use the M formula: Text.BeforeDelimiter([CustomerProduct], " ", {0, RelativePosition.FromEnd}) for Customer, and Text.AfterDelimiter([CustomerProduct], " ", {0, RelativePosition.FromEnd}) for Product (when Product is one word). For two-word Product names split from the end twice.

06 — Cleaning

Step 3 — Clean Up: Blanks, Trim, Case, Types

Apply these transformations to the full table once you have your 5 columns.

Remove blank and "(blank)" rows

  1. Filter out fully blank rows. Click the drop-down on the Customer column. Untick (null) and (blank). Click OK.

  2. Remove rows where Customer is the literal text "(blank)". Home → Remove Rows → Remove Blank Rows handles nulls. For literal "(blank)" strings, use Home → Replace Values: replace (blank) with nothing (leave "Replace With" empty), then use Remove Blank Rows again.

Trim & Clean whitespace

  1. Select all text columns (Customer, Product, Region). Hold Ctrl and click each header.

  2. Apply Trim. Transform tab → Format → Trim. This removes leading and trailing spaces.

  3. Apply Clean. With the same columns still selected: Transform → Format → Clean. This removes non-printable characters (common in pasted text).

Standardise text case

  1. Customer & Product — Capitalize Each Word. Select the Customer and Product columns. Transform → Format → Capitalize Each Word. This ensures "john smith" becomes "John Smith".

  2. Region — Capitalize Each Word. Same steps for Region to normalise "north" → "North".

Convert Amount to a number

  1. Remove currency symbols if present. Select the Amount column. Transform → Replace Values. "Value to Find": £ (or $). "Replace With": leave empty. Click OK.

  2. Remove commas (thousands separators) if present. Repeat Replace Values: find , → replace with nothing.

  3. Change type to Decimal Number. Click the type icon on the Amount column header → Decimal Number. Check for red error markers — click into any error to see the offending value.

Parse dates

  1. Change type to Date. Click the Date column's type icon → Date. Power Query detects ISO 8601 (YYYY-MM-DD) automatically.

  2. For non-ISO formats (e.g. DD/MM/YYYY). Right-click the Date column header → Change Type → Using Locale. Data Type: Date. Locale: English (United Kingdom) for DD/MM/YYYY. Click OK.

  3. Extract date parts (optional). With Date column selected: Add Column → Date → Year / Month / Day — useful for pivot tables or further grouping.

✔ Checkpoint
At this point your Applied Steps pane should list something like: Source → Promoted Headers → Changed Type → Filtered Rows → Trimmed Text → Cleaned Text → Capitalised Text → Replaced £ → Changed Type (Amount) → Changed Type (Date). Each step is undoable — click any step to review its output.
07 — Finish

Step 4 — Close & Load to a New Worksheet

  1. Review the final preview. In the Power Query Editor, scroll through the data preview to confirm all 5 columns look correct, no stray null rows remain, and types show the correct icons (ABC for text, 1.2 for decimal, 📅 for date).

  2. Rename the query (optional but recommended). In the Query Settings pane on the right, change "Name" from Table1 to something descriptive like SalesData_Clean.

  3. Close & Load. Click Home tab in Power Query → Close & Load drop-down → Close & Load To….

  4. Choose destination. In the "Import Data" dialog:

    • Select Table (not PivotTable or Connection Only)
    • Select New worksheet
    • Click OK

  5. Verify the output sheet. Excel creates a new worksheet (e.g. Sheet2) with an Excel Table containing your 5 clean columns. The table is linked to the Power Query — right-click anywhere in it and choose Refresh to re-run all steps whenever the source data changes.

ℹ Refreshing automatically
To refresh on file open: go to Data tab → Queries & Connections, right-click your query → Properties → tick "Refresh data when opening the file".

🎯 You now have a repeatable, refreshable cleaning pipeline

Every transformation is recorded as a step in the Applied Steps pane. When the source data changes, simply hit Refresh and Power Query re-runs the full pipeline — no manual re-cleaning needed.
