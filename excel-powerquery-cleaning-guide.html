Data Cleaning Guide

A practical, step-by-step reference for cleaning pasted data in Excel before loading it into Power Query — covering safe paste methods, removing blank columns and rows, trimming trailing delimiters, and troubleshooting the most common issues that cause extra columns to appear unexpectedly.

01 — Background

Why Blank Columns Appear

When you paste data into Excel — especially data copied from a browser, PDF, or another application — Excel can silently create one or more empty columns to the right of your data. These ghost columns look invisible, but Power Query detects them and imports them as unnamed columns (Column1, Column2, …), which breaks query steps and causes downstream errors.

The most common causes are:

Root Cause What Happens in Excel Blank Cols?
Pasting with formatting Clipboard includes invisible trailing whitespace or tab characters from the source app Yes
Trailing delimiters / tabs CSV or TSV data with a delimiter at the end of each row creates an extra empty field Yes
Multiple consecutive spaces Excel may split on spaces if Text-to-Columns is used with space as delimiter Yes
Selecting entire columns If you select column A:Z instead of only used cells, every empty column is included in the Table range Yes
Paste Special → Values only Strips formatting but does not strip tab/space characters embedded in the text Maybe
Paste via Notepad (clean text) Notepad strips rich-text formatting; only plain text is transferred to clipboard Rarely
ℹ️
Key Point

Power Query reads the Table range or the Used Range of a sheet exactly as Excel reports it. If Excel thinks column F is used (even if it just holds a space character), Power Query will include it.

02 — Best Practices

Best Practices for Pasting Data

The safest approach is to strip all rich-text formatting before the data reaches Excel. Two methods reliably achieve this:

A

Method A — Paste via Notepad (Recommended)

Copy your data from the source (Ctrl + C), open Notepad (Win + R → type notepad → Enter), paste with Ctrl + V, then copy again from Notepad (Ctrl + A → Ctrl + C), and finally paste into Excel. Notepad strips all formatting, leaving only plain text — this eliminates most hidden characters.

B

Method B — Paste Special → Text / Values

In Excel, instead of Ctrl + V, use Ctrl + Alt + V (or Home → Paste → Paste Special), then choose "Text" (or "Unicode Text" / "Values"). This prevents Excel from importing hidden styles from the clipboard. Note: this does not remove embedded tab or space characters within the data itself.

💡
Pro Tip

For recurring data imports, consider saving the source as a .csv file and importing it directly into Power Query via Get Data → From Text/CSV. Power Query handles the delimiter parsing correctly and avoids the clipboard entirely.

⚠️
Common Mistake

Do not use "Paste Special → Keep Source Formatting" when the source is a browser table or PDF. These sources embed invisible tab stops and line breaks that will create phantom columns in Excel.

03 — Range Selection

Selecting the Exact Used Range

When you convert a block of cells to an Excel Table, Excel uses the selection you made — not just the cells that contain visible data. If you selected full columns (e.g. A:E) or accidentally included empty columns, those columns become part of the Table.

Correct approach:

1

Click a single cell inside your data block

Do not pre-select a large range. Excel can detect the data boundary automatically when you insert a Table from a single cell within the data.

2

Use Ctrl + End to verify the last used cell

Press Ctrl + End — Excel jumps to the last cell in the Used Range. If this lands far to the right of your data (e.g. column Z when your data ends at column E), there are stray characters in those extra columns. Delete them before creating your Table.

3

If inserting a Table, let Excel auto-detect or select precisely

Press Ctrl + T with the cursor inside a data cell. Excel will highlight the contiguous block. Verify the selection boundary in the dialog before clicking OK. Adjust if it includes extra columns.

🚫
Never Do This

Do not select entire columns (e.g. clicking column headers A, B, C) when creating a Table. This makes the Table range include every row in the spreadsheet — over 1 million rows — and will include all empty columns in that column range in Power Query.

04 — Delimiters

Trimming Trailing Delimiters

If your source data is comma-separated (CSV) or tab-separated (TSV), rows that end with a trailing delimiter — for example John,Smith,100, — will produce an extra empty column when parsed. The trailing comma or tab tells the parser "there is one more field after this" — even though that field is empty.

Before pasting into Excel, clean the source text:

1

Inspect and strip trailing commas / tabs in Notepad or a text editor

Open the source file in Notepad++ or VS Code. Use Find & Replace with a regular-expression pattern to strip trailing delimiters before copying. For trailing commas: find ,\s*$ (regex mode) and replace with nothing.

2

Or: use Excel's Text-to-Columns to control delimiter parsing

Paste the raw text into a single Excel column first, then use Data → Text to Columns. In the wizard, choose your delimiter. Do not tick "Treat consecutive delimiters as one" unless your data is space-delimited — that setting hides the problem rather than fixing it.

3

Or: use Power Query's own delimiter handling

If importing a CSV via Get Data → From Text/CSV, Power Query shows a preview. If you see an unnamed column at the right, go to Transform → Remove Columns and remove it in Power Query Editor rather than in Excel. This is cleaner for repeatable refreshes.

05 — Empty Cells

Removing Empty Columns & Rows

Before you open Power Query, empty columns and rows should be removed from your Excel sheet. Even a single space character in an otherwise blank column is enough for Excel to include it in the Used Range.

1

Find and delete blank columns to the right of your data

Select the column immediately to the right of your last data column. Hold Ctrl + Shift + Right Arrow to extend the selection to the last column (XFD). Right-click → Delete. This removes all columns to the right, resetting the Used Range boundary.

2

Find and delete blank rows below your data

Select the row immediately below your last data row. Hold Ctrl + Shift + Down Arrow to extend to row 1 048 576. Right-click → Delete. Then save the file — this forces Excel to recalculate the Used Range.

3

Save and recheck with Ctrl + End

Press Ctrl + S to save, then press Ctrl + End again. If it still jumps beyond your data, there are cells with space characters. Select those columns/rows, press Delete (not Backspace), then save again.

4

Optional: use Find & Select to catch hidden spaces

Go to Home → Find & Select → Go To Special → Blanks. Excel selects every blank cell in the current selection. If you see selections outside your expected data range, those cells contain nothing visible but may have whitespace. Press Delete to clear them.

💡
Why Save Matters

Excel only recalculates the Used Range (Ctrl+End destination) when the workbook is saved. Deleting columns and then immediately opening Power Query without saving first may still show the old boundary.

06 — Excel Table

Converting to an Excel Table Safely

Converting your data to a named Excel Table (Ctrl + T) before loading into Power Query gives you cleaner refresh behaviour and makes column names explicit. Follow these steps in order:

1

Paste your data using Method A or B (see Section 2)

Start clean — use Notepad or Paste Special → Text.

2

Remove blank columns and rows (see Section 5)

Do this before creating the Table. It is much harder to remove ghost columns after the Table is defined.

3

Click one cell inside the data → Ctrl + T

In the "Create Table" dialog, verify the range shown matches your actual data range exactly. Check "My table has headers" if your first row is a header row.

4

Give the Table a meaningful name

Click anywhere in the Table → Table Design tab → change "Table1" to something descriptive like SalesData or OrdersRaw. Power Query will reference this name, making your query easier to read and maintain.

5

Save before opening Power Query

Press Ctrl + S. Now open Power Query via Data → Get Data → From Table/Range. You should see only the columns you expect — no unnamed extras.

⚠️
Column Header Tip

Make sure every header cell in row 1 is unique and non-blank. If two columns share a name or a header is empty, Power Query will rename them automatically (e.g. Column1), which can break downstream steps.

07 — Troubleshooting

Troubleshooting Extra Columns

Even after careful preparation, extra columns can still appear in Power Query. Here are the most common scenarios and how to fix them:

🔍

Problem: Power Query shows "Column1", "Column2" at the right

Unnamed columns appear at the end of the query, even though Excel looks clean.

✔ Fix

Close Power Query and go back to Excel. Press Ctrl + End — if the cursor lands beyond your data, select all columns from your last data column to the cursor column, right-click, and Delete. Save (Ctrl + S) and re-open Power Query. The extra columns should be gone.

🔍

Problem: Extra columns appear because of multiple spaces in the data

Data was split using "Space" as a delimiter in Text-to-Columns, and fields with two or more consecutive spaces created blank intermediary columns.

✔ Fix

In the Text-to-Columns wizard, check "Treat consecutive delimiters as one". Alternatively, before splitting, use Find & Replace (Ctrl + H) to replace double-spaces with a single space. Repeat until no double-spaces remain, then re-run Text-to-Columns.

🔍

Problem: Selecting entire columns (e.g. A:E) created the Table

The Excel Table range spans all rows, and Power Query includes thousands of blank rows and any empty columns in the range.

✔ Fix

Delete the Table (Table Design → Convert to Range), then recreate it: click a single cell inside the data, press Ctrl + T, and verify the range in the dialog. Never select whole columns before pressing Ctrl + T.

🔍

Problem: Blank columns reappear after every paste / data refresh

The data source consistently includes trailing tabs or commas, so the issue recurs each time.

✔ Fix

Handle it once in Power Query instead of Excel: in Power Query Editor, select the blank columns (hold Ctrl to select multiple), right-click → Remove Columns. This removal step runs automatically on every refresh, so you never need to clean the Excel sheet manually again.

🔍

Problem: Blank columns contain a single space and are invisible

The column looks empty but Ctrl + End still moves past it — it contains a space character.

✔ Fix

Select the suspicious columns. Use Home → Find & Select → Replace (Ctrl + H): in "Find what", type a single space; leave "Replace with" empty; click Replace All. Then delete the now-truly-empty columns and save.

🔍

Problem: Power Query adds "Changed Type" step that references wrong columns

After removing blank columns, old query steps still reference the original column positions and throw errors.

✔ Fix

In Power Query Editor, expand the Applied Steps pane. Delete the step named "Changed Type" (or any step that references the now-deleted column). Then re-apply a new "Changed Type" step by selecting all columns and using Transform → Detect Data Type.

08 — Checklist

Quick Pre-Load Checklist

Run through this checklist every time before opening Power Query on pasted data:

  • Pasted using Notepad or Paste Special → Text — not "Keep Source Formatting"
  • Trailing delimiters removed from CSV/TSV source before pasting
  • No consecutive spaces if space is used as a delimiter in Text-to-Columns
  • Ctrl + End lands on the correct last cell — not beyond the actual data
  • All columns to the right of data are deleted — not just cleared
  • All rows below data are deleted — not just cleared
  • File saved (Ctrl + S) after deletions to reset Used Range
  • Table created from a single cell inside data — not from a full-column selection
  • Table name is descriptive and every header is unique and non-blank
  • If blank columns still appear in Power Query, handle them with a "Remove Columns" step in the query rather than repeatedly cleaning Excel manually
🚀
Interview-Ready Insight

If asked about data quality in Excel or Power Query, mention that you always verify the Used Range with Ctrl + End before creating a Table, use Paste Special → Text or Notepad to strip formatting, and handle recurring blank-column issues directly in Power Query's Applied Steps for repeatable, self-cleaning refreshes. This demonstrates structured, professional data hygiene.
